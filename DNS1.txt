Что такое DNS (Domain Name System).
Доменная система имен (Domain Name System, DNS)  - это база данных, которая ведет себя, как записная книга. В ней хранится основная информация относительно каждого зарегистрированного компьютера в сети : его имя, IP-адрес и данные для маршрутизации.
Данная база необходима для  соотнесения IP-адресов устройств в сети и преобразовании численного IP -адреса в привычное символьное имя для более легкого запоминания. Благодаря этому мы можем совершать  запрос в интернете любого сайта.  У DNS сервера  иерархическая структура хранения данных,которая напоминает дерево с раскидистыми ветвями. Для того чтобы открыть нужный сайт по запросу в поиске  будет произведен  запрос к домену первого уровня .ru, к домену второго уровня названиесайта.ru, домену третьего уровня panel.названиесайта.ru и т. д пока не найдет нужный сайт в одной из ветвей сервера.

Различие протоколов HTTP/HTTPS и их версий.
Вся информация в Интернете передается с помощью определенных протоколов. Первым был разработан протокол HTTP (прокотол передачи гипертекса)  в 1989 году. HTTP представляет собой набор правил и стандартов, которые регулируют способ передачи  информации в сети. HTTP предоставляет стандартные правила для взаимодействия веб-браузеров и серверов.  
Если описать простым языком - это как отправить письмо без конверта и надеяться на то что почтальон будет придерживаться правил секретности и безопастности и не прочитает письмо.
После того,как стали распространяться онлайн-платежи, программисты задумались о том, чтобы сделать передачу данных более безопасной,тогда был разработан прокотол HTTPS.
HTTPS - это защищенный протокол передачи гипертекста. Он позволяет создавать защищенное зашифрованное соединение между сервером и браузером. Так же обеспечивает двунаправленную безопасность данных. Это помогает защитить потенциально конфиденциальную информацию от кражи.
Шифрование происходит за счет сертификата SSL . Благодаря этому сертификату данные на сервер отправляются в зашифрованном виде и по пути его будет сложно перехватить мошенникам.
Принципиальные отличия протоколов заключаются в том что:
1) В HTTP отсутствует шифрование отправляемых данных, а HTTPS использует сертификаты защиты данных SSL /TLC.
2)HTTP работает на уровне приложения, а HTTPS - на транспорном уровне. 
3) HTTP по умолчанию работает по 80-ому порту, а HTTPS – работает через 443 порт.
4) HTTP передает данные открытым текстом, а HTTPS - зашифрованным.
5) HTTP по сравнению с HTTPS работает быстрее, поскольку последнему нужно время для шифрования канала связи.

На данный момент есть четыре стандарта HTTP протокола:
1)Стандарт HTTP/0.9 – версия протокола HTTP 0.9
Он разработан для облегчения доступа и создания навигации при помощи гипертекста. Протокол версии 0.9 был призван упорядочить взаимодействие между клиентом и сервером в сети. После появления стандарта HTTP/0.9 появилось разделение функций между клиентом и сервером при их взаимодействии. Стандарт HTTP/0.9 содержит в себе основы синтаксиса и семантики протокола HTTP.
2)Позже был выпущен информационный документ RFC 1945 (стандарт HTTP/1.0). Данный документ стал основой для реализации приложений и компонентов с использованием протокола HTTP версии 1.0.
3) Следующим была выпущена версия протокола HTTP1: был разработан стандарт HTTP/1.1 и описан он в документе RFC 2068. Основным нововведением в версию протокола HTTP 1.1 был режим постоянного , другими словами: за одно соединение можно было отправлять несколько HTTP запросов и получать несколько HTTP ответов в том порядке, в котором делались запросы. Вторым основным нововведение в версию протокола HTTP 1.1 является то, что теперь клиент при установке соединения с сервером должен обязательно посылать имя хоста в специальном поле HTTP заголовка . 
4) Последней была разработана  финальная версия черновика протокола HTTP 2, это еще не стандарт, но черновик  «показывает» куда будет двигаться развитие интернета. Версия протокола HTTP 2 является бинарной. В версии протокола HTTP 2 будет поддерживаться мультиплексирование (объединение) запросов, поскольку появится объединение, появится и приоритет запросов и многое другое.
